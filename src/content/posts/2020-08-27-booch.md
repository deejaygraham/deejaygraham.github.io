Dropping some photos here for some resources I used when first getting into programming at a time when Object Orientation 
was a fairly new thing. 

## Object Oriented Analysis

Learning about OO programming and design, Grady Booch's book "Object Oriented Analysis And Design" was a required reading resource. 

<img src="/assets/img/posts/booch/object-book-front.jpg" alt="Book Cover Front" />

<img src="/assets/img/posts/booch/object-book-back.jpg" alt="Book Cover Back" />

Anyone who has an original copy of this book will be familiar with the object notation used in the diagrams. On top of that, 
there was Jim Rumbaugh's notation for classes, Peter Coad's notation, and practically anyone else with an idea about how to 
build classes. Which brings me to the next artefact...


## UML 

A little later, UML came along amalgamating several of the diagramming methods into a "clean" specification and/or 
de-facto standard. I recall downloading the UML pdf specs when I first had a machine with an internal dial-up modem 
on my work computer but resources like this from Rational helped to make the case for why UML was a better idea.
